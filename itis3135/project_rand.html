Project Requirements and Specifications

Project Overview

This website is intended to advertise the artwork of my sister, Bridget King. This site will contain an interactive gallery of her artwork including prices and an Etsy store link.

This website is intended to attract customers to Bridget's Etsy store. The target audience of this site will be people looking to decorate their home or office.

The main function of the site will be to catalog and display Bridget's artwork for sale, but the site will also include information about Bridget and mental healthcare resources related to her work.

Client Information

The client's name is Bridget King.

This site is associated with Bridget's Etsy store.

The client's email is bridget2220@yahoo.com

Page Design

Home Page

  1. The purpose of this page is to be a visually interesting landing page where the potential user can see what kind of art they will be browsing.
  2. The audience of this page will be new visitors or new customers.
  3. The content in this page will mostly be images, but it will also contain other links relevant to the site.
  4. This page will not take any user input other than mouse clicks.
  5. No data validation needed.
  6. This page will contain hyperlinks to other relevant pages.
  7. This page will display popular items from the gallery, and some other introductory information.

About the Artist

  1. This page is meant to familiarize customers with the artist of the gallery paintings.
  2. The audience of this page will be customers who want to learn more about the artist they are supporting.
  3. The page will contain some pictures of the artist, her work, and her pets. It will also contain text information about the artwork.
  4. This page will not take any user input other than mouse clicks.
  5. No data validation needed.
  6. This page will contain both hyperlinks and dropdown menus.
  7. The images on the page will change if you click through the bio pictures.

Gallery

  1. The purpose of this page is to display all of the artwork and prices in a coherent menu, and to search through artwork.
  2. This page is for the customers who want to buy or browse the artwork.
  3. The page will contain a grid of all the artwork and it's associated prices.
  4. The user will be able to search for artwork by name on this page
  5. Inputs will have to verified as non empty strings.
  6. The page will contain buttons that link to the Etsy store, or to a product page.
  7. The page should dynamically resize as users browse through the images.

Mental Health Resources

  1. The purpose of this page is to promote mental health of customers or users, because it is something Bridget is passionate about.
  2. The audience of this page might be anyone who is looking for mental health resources.
  3. The page will contain information on mental health resources by region.
  4. This page will not take any user input other than mouse clicks.
  5. No data validation needed.
  6. This page will contain hyperlinks for resources, and implement a dropdown to select what resources are displayed on the page.
  7. This page will be used to navigate users to other pages.

Charity

  1. The purpose of this page is to provide information on charities that Bridget supports.
  2. The audience of this page is generous customers who might visit the site looking for artwork.
  3. The page will contain links to donation information from charities that Bridget supports.
  4. This page will not take any user input other than mouse clicks.
  5. No data validation needed.
  6. This page will contain both hyperlinks and dropdowns.
  7. The page will be used to redirect the user to donate to various charity websites.

Dynamic Functionality

There are many dynamic functions I want to implement in this site. I want to create a dynamic gallery of images for the gallery page, and a search bar for finding specific artwork. This functionality would be similar to a webstore like Amazon. Additionally Javascript would be used to dynamically change the information displayed on each page depending on what the user clicked.
